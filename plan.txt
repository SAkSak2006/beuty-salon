 План: ИС "Салон красоты" с Telegram чат-ботом

 Контекст

 Дипломный проект (09.02.07, Мегионский политехнический колледж, защита лето 2026). Существует рабочий прототип CRM на
 Vanilla JS + Express + SQLite с полным функционалом (дашборд, записи, клиенты, сотрудники, услуги, склад, отчёты). Нет
  авторизации и чат-бота. Цель: превратить в полноценную ИС с PostgreSQL + Redis + RabbitMQ + Telegram бот с Mini App
 для бронирования.

 Архитектура

 beauty-salon/
 ├── docker-compose.yml          # PostgreSQL + Redis + RabbitMQ
 ├── .env
 ├── crm-frontend/               # Существующий фронтенд (Vanilla JS) + логин
 ├── api-server/                  # Express + PostgreSQL + JWT + Redis + RabbitMQ
 ├── telegram-bot/                # grammy бот (кнопки + Mini App)
 └── mini-app/                    # Vue 3 + Vite (веб-приложение внутри Telegram)

 Стек: Node.js/Express, PostgreSQL, Redis, RabbitMQ, grammy, Vue 3 + Vite, Docker Compose

 Упрощения при нехватке времени

 1. RabbitMQ → заменить на SELECT ... FOR UPDATE в PostgreSQL (та же защита от гонок)
 2. Redis → заменить на Map в памяти с TTL (показывает концепцию)
 3. Vue Mini App → Vanilla JS + Telegram Web App SDK (проще)
 4. Роль master → реализовать последней, начать с admin/owner

 Верификация

 1. docker-compose up -d → PostgreSQL, Redis, RabbitMQ запущены
 2. cd api-server && npm start → CRM API на порту 3000
 3. Открыть crm-frontend/login.html → войти как admin → все разделы работают
 4. Открыть Telegram бот → /start → кнопки работают
 5. "Записаться" → Mini App → выбрать услугу → мастера → дату → время → подтвердить
 6. В CRM появилась новая запись
 7. За 2 часа до записи — уведомление в Telegram
